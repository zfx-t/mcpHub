MCPHub

一个可自托管的 MCP 中间件平台。你可以用本地插件把已有 REST API、网站能力、 管理后台接口和可信工作流转换为 Agent 可发现、可审计、可调用的 MCP resources 与 tools。

它不替代原系统，也不是单个业务插件；它提供一套可复制的适配层，让没有 MCP 的能力被标准化暴露给 Agent。

About

把一次性的 API 使用说明，沉淀成可复用的 MCP 接口。

很多后台、SaaS 和网站已经有可用 API 或业务流程，但它们没有 MCP。MCPHub 的价值是把这些能力封装成稳定、 可验证、可审计的 MCP resources 与 tools，让 Agent 不再依赖散落在 prompt 或 skill 里的临时说明。

Why

为什么需要它

直接把 API 文档塞给 Agent 容易失控：鉴权、参数、错误处理、审计和多步骤流程都难以复用。

What

它是什么

MCPHub 是服务端插件运行时。插件描述能力，平台负责加载、凭据、策略、审计、诊断和 MCP 暴露。

Not

它不是什么

它不是爬虫专用项目，不是单个 B 站或后台插件，也不接管上游系统自己的权限和业务规则。

Architecture

项目结构是一条从插件到 MCP 的执行链。

开发者编写本地插件，MCPHub 加载并校验它，再通过 gateway 把资源和工具暴露给 Agent。 凭据、策略和审计在调用链中统一处理。

apps/server Fastify HTTP API, health, diagnostics, Streamable HTTP `/mcp`
packages/plugins local loader, plugin manifest, standard validation
packages/mcp gateway, SDK server, resources/tools, executor runtime
policy + audit + credentials effect control, call evidence, environment-backed secrets
apps/web independent static project portal deployed with GitHub Pages
Quick Start

启动服务，检查 `/mcp`，再生成第一个插件。

这组命令覆盖当前 dev 版本最小闭环：安装、启动、MCP inspect、创建插件、验证插件。

terminal 
pnpm install
REQUEST_LOGGING=false pnpm dev
pnpm mcp:client --url http://127.0.0.1:3000/mcp inspect
pnpm plugin:create my-admin --template http-api --tool-name my.admin.users.list
pnpm plugin:verify examples/plugins/my-admin

Plugin System

简单 API 用 HTTP operation，复杂业务用 executor workflow。

插件和平台解耦：插件负责业务适配，平台提供受控执行上下文、凭据解析、策略判断、审计记录和统一 MCP 出口。

HTTP operation

适合一次 REST 请求即可完成的读取或写入操作。

Executor workflow

适合校验、上传、提交、轮询、清洗和聚合这类多步骤流程。

plugin shape 
examples/plugins/my-plugin/
|-- index.js
|-- plugin.config.json
|-- README.md

export default {
  id: "my-admin",
  version: "0.1.0",
  mcphub: {
    minVersion: "0.1.0",
    capabilities: ["http", "credentials", "policy"]
  },
  tools: [{
    name: "my.admin.users.list",
    effect: "read",
    operation: { type: "http", method: "GET", path: "/api/users" }
  }]
};
Progress

当前 dev 版本已经能跑，后续重点是降低适配成本。

这里明确区分已经完成的能力和下一阶段计划，避免把路线图当作已交付功能。

Available now

  • Streamable HTTP MCP endpoint
  • Web content resources and tools
  • Local plugin loading
  • HTTP operation tools
  • Executor workflow tools
  • Credential, policy, and audit support
  • Plugin standard validation
  • Generic MCP client CLI
  • Dev deployment and diagnostics docs

Next priorities

  • API/OpenAPI 文档导入，生成可编辑的 MCPHub 插件骨架。
  • 更完整的插件生命周期诊断和运行状态可见性。
  • 部署安全加固，以及生产暴露边界说明。
  • 当单页门户不够用时，再拆分为更完整的 docs site。
